Overview

Building an ordering platform end-to-end

This project involved designing and developing a complete database-driven ordering system for a bakery — from relational database design through to a customer-facing, responsive UI. Every layer was built from scratch: no CMS, no frameworks, no shortcuts.

The system handles the full customer journey — browsing, ordering, user account management — while giving administrators a separate panel for order management, product updates, and reporting.

Features Built

What the system does

Customer Ordering

Browse products by category, add to cart, apply quantities, and place orders with real-time stock validation and instant confirmation.

User Authentication

Secure registration, login, and session management with hashed passwords, CSRF protection, and role-based access for customers vs admins.

Admin Panel

Dedicated backend for managing orders, updating product listings and pricing, viewing order history, and generating simple sales summaries.

Database Design

Relational MySQL schema with normalised tables for users, products, orders, and order_items — supporting efficient queries and referential integrity.

Responsive UI

Fully mobile-responsive front end built in pure HTML/CSS/JS — no frameworks. Designed to work cleanly across screen sizes without degradation.

Order Tracking

Customers can view current order status through their account dashboard. Admins can update order stages (received, baking, ready, delivered) in real time.

Technical Approach

How it was architected

The application was built on a PHP MVC-style structure — routing incoming requests through a central controller, with clean separation of data logic, business rules, and presentation templates. This made the codebase maintainable and easy to extend without full rewrites.

MySQL handled all persistent storage, with prepared statements throughout the PHP layer to prevent SQL injection. Sessions were managed server-side, with role flags stored in session data to control view rendering and admin access permissions.

The front end was written in semantic HTML5 and CSS3 with vanilla JavaScript for cart interactions and form validation — deliberately avoiding framework dependencies to keep the project lightweight and fully within my control.

End-to-end full-stack delivery

The completed system demonstrated the ability to conceive, design, and ship a multi-user web application independently — handling database architecture, server-side logic, authentication, and responsive UI within a single cohesive codebase. It remains the clearest proof of full-stack capability in this portfolio.
